SourceTrak Integration API

The IBP SourceTrak API allows one to request dynamic tracking numbers, log page visits and heartbeats via a RESTful service.

Get Number

The default request is made via GET and is used to get a phone number to display on one's site. All request parameters should be URL encoded. Requests are made to: https://st1.ifbyphone.com/st/

GET Parameters

    
        api=true
        _stk=PUBLIC_KEY
        dr=DOCUMENT_REFERER
        dl=DOCUMENT_LOCATION
        dt=DOCUMENT_TITLE

The api parameter is always set to true to indicate this is an external API request. The _stk is used to authorize one's API request. The dr parameter is the document referer which is used to match against one's SourceTrak setup pertaining to what web site the user came from. The dl paramater is used to indicate the current domain and query string parameters. The dt parameter is used to set the title of the page view for reporting.

Example Request

https://st1.ifbyphone.com/st/?api=true&_stk=6c9a4a044we846f80gj55c9cb7e6a72964e6da82d3&dr=http%3A%2F%2Fgoogle.com&dl=http%3A%2F%2Fwww.oursite.com%3Fst-t%3D50%20Off%20Campaign&dt=50%25%20Off%20Landing%20Page

Example Response

    
        {
          "bid": "d56ad1a2-8dbf-98ab-1f12-4ed429e43e3d",
          "pid": "95acfe77-8bd6-bf42-b6a0-3abe1fcd9156",
          "stn": "3129975796",
          "ntr": "5555555555",
          "rc": "phone",
          "rid": "",
          "se": 600,
          "he": 10800,
          "h": "30",
          "a": 0,
          "m": "8662571076",
          "sgn": 1,
          "integrations": []
        }

bid: Browser ID to be associated with any request from that user's browser. This ID domain agnostic
pid: Property ID identifying the request as pertaining to this domain set browser ID combination
stn: SourceTrak number
ntr: Number to replace
rc: Replacement HTML Class
rid: Replacement HTML ID
se: Soft Timeout (seconds)
he: Hard Timeout (seconds)
h: Heartbeat (seconds)
a: Flag for wrapping a tags around the phone number on mobile
m: Main number
sgn: Flag for if the SourceTrak number is a SourceGuard number
integrations: List of associated integrations

Log Page Visit

The log request is made via GET and is used to log an end-user's page request activity on one's web site. All request parameters should be URL encoded. Requests are made to: https://st2.ifbyphone.com/st/log

GET Parameters

    
        api=true
        _stk=PUBLIC_KEY
        dr=DOCUMENT_REFERER
        dl=DOCUMENT_LOCATION
        dt=DOCUMENT_TITLE
        bid=BROWSER_ID
        pid=PROPERTY_ID

Example request

https://st2.ifbyphone.com/st/log?api=true&_stk=6c9a4a044we846f80gj55c9cb7e6a72964e6da82d3&dr=http%3A%2F%2Fgoogle.com&dl=http%3A%2F%2Fwww.oursite.com%3Fst-t%3D50%20Off%20Campaign&dt=50%25%20Off%20Landing%20Page&bid=d56ad1a2-8dbf-98ab-1f12-4ed429e43e3d&pid=95acfe77-8bd6-bf42-b6a0-3abe1fcd9156

Example Response

    Status Code:200 OK

A successfull page visit request responds with an empty 200.

Log Heartbeat

The heartbeat request is made via GET and is used to log if the end-user still has a page open in the browser. The heartbeat requests are made to the same end-point as logging page visits with the param of _stb=1. These requests are used to rank an inbound calls accuracty to be tracked back to an end-user's history. All request parameters should be URL encoded. Requests are made to: https://st2.ifbyphone.com/st/log

GET Parameters

    
        api=true
        _stk=PUBLIC_KEY
        _stb=1
        dr=DOCUMENT_REFERER
        dl=DOCUMENT_LOCATION
        dt=DOCUMENT_TITLE
        bid=BROWSER_ID
        pid=PROPERTY_ID

Example request

https://st2.ifbyphone.com/st/log?api=true&_stk=6c9a4a044we846f80gj55c9cb7e6a72964e6da82d3&_stb=1&dr=http%3A%2F%2Fgoogle.com&dl=http%3A%2F%2Fwww.oursite.com%3Fst-t%3D50%20Off%20Campaign&dt=50%25%20Off%20Landing%20Page&bid=d56ad1a2-8dbf-98ab-1f12-4ed429e43e3d&pid=95acfe77-8bd6-bf42-b6a0-3abe1fcd9156

Example Response

    Status Code:200 OK

A successfull heartbeat request responds with an empty 200.
